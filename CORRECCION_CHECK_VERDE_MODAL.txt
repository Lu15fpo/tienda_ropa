🐛 PROBLEMA IDENTIFICADO:

En el modal de selección de métodos de pago (checkout):

COMPORTAMIENTO INCORRECTO:
1. Al abrir el modal, el check verde ✓ aparece en el método predeterminado
2. Al seleccionar OTRO método, el check verde NO se mueve
3. El check verde permanece en el método anterior
4. PERO el método SÍ cambia en la pantalla principal

CAUSA:
El check verde se mostraba basándose en `method.isDefault` (campo en Firebase),
pero al seleccionar un método en el modal, solo se actualiza
`selectedPaymentMethod.value` localmente, NO se cambia `isDefault` en Firebase.

Resultado: El check verde sigue en el que tiene `isDefault: true` en Firebase.

═══════════════════════════════════════════════════════════════════════════════

✅ SOLUCIÓN IMPLEMENTADA:

CAMBIO EN: checkout_controller.dart

ANTES:
```dart
return ListView.builder(
  itemBuilder: (_, index) {
    final method = paymentMethods[index];
    return ListTile(
      // ...
      trailing: method.isDefault  // ← Basado en Firebase
          ? const Icon(Icons.check_circle, color: Colors.green)
          : null,
    );
  },
);
```

DESPUÉS:
```dart
return ListView.builder(
  itemBuilder: (_, index) {
    final method = paymentMethods[index];
    return Obx(  // ← Agregado Obx para reactividad
      () => ListTile(
        // ...
        trailing: selectedPaymentMethod.value.id == method.id  // ← Basado en selección actual
            ? const Icon(Icons.check_circle, color: Colors.green)
            : null,
      ),
    );
  },
);
```

📝 NOTA IMPORTANTE:

Este cambio NO afecta el campo `isDefault` en Firebase.
- isDefault en Firebase: Marca el método predeterminado para futuras sesiones
- selectedPaymentMethod: Método seleccionado para la compra ACTUAL

Son dos conceptos diferentes:
• isDefault = Método predeterminado general
• selectedPaymentMethod = Método para esta compra específica

El check verde ahora muestra correctamente el método que se usará en ESTA compra.
