CAUSA RAÍZ #1: Queries faltantes en AndroidManifest
----------------------------------------------------
Desde Android 11+, las apps deben declarar explícitamente
qué "intents" pueden consultar debido a Package Visibility.

PROBLEMA:
• `canLaunchUrl()` consulta si hay apps para manejar HTTPS
• AndroidManifest NO tiene queries para scheme "https"
• Android retorna: NO HAY APPS (aunque el navegador existe)
• `canLaunchUrl()` retorna FALSE
• NO se intenta abrir

CAUSA RAÍZ #2: Validación innecesaria
--------------------------------------
`canLaunchUrl()` a veces retorna false incorrectamente,
pero `launchUrl()` SÍ puede abrir la URL de todas formas.

SOLUCIÓN: Intentar abrir SIN validar primero.

═══════════════════════════════════════════════════════════════
  ✅ SOLUCIÓN #1: QUERIES EN ANDROIDMANIFEST
═══════════════════════════════════════════════════════════════

ARCHIVO: android/app/src/main/AndroidManifest.xml

AGREGADO:
```xml
<queries>
    <!-- Ya existente -->
    <intent>
        <action android:name="android.intent.action.PROCESS_TEXT"/>
        <data android:mimeType="text/plain"/>
    </intent>

    <!-- 🔧 NUEVO: Queries para url_launcher -->
    <intent>
        <action android:name="android.intent.action.VIEW" />
        <data android:scheme="https" />
    </intent>
    <intent>
        <action android:name="android.intent.action.VIEW" />
        <data android:scheme="http" />
    </intent>
</queries>
```

EXPLICACIÓN:
------------
• Declara que la app puede consultar navegadores
• `canLaunchUrl()` ahora puede ver los navegadores instalados
• Android retorna: SÍ HAY APPS (navegadores)

═══════════════════════════════════════════════════════════════
  ✅ SOLUCIÓN #2: LANZAR SIN VALIDAR
═══════════════════════════════════════════════════════════════

ARCHIVO: lib/features/shop/controllers/product/order_controller.dart

NUEVA LÓGICA:
```dart
// Intentar abrir PDF directamente SIN validar con canLaunchUrl
try {
  final uri = Uri.parse(pdfUrl);

  // Intento #1: externalApplication
  final launched = await launchUrl(
    uri,
    mode: LaunchMode.externalApplication,
  ).catchError((error) async {
    // Intento #2: platformDefault (navegador)
    return await launchUrl(
      uri,
      mode: LaunchMode.platformDefault,
    );
  });

  if (!launched) {
    // Mostrar error
  }
} catch (e) {
  // Intento #3: Último recurso
  await launchUrl(
    uri,
    mode: LaunchMode.platformDefault,
  );
}
```

ESTRATEGIA DE 3 NIVELES:
------------------------
1. Intenta con externalApplication (app de PDF)
2. Si falla, intenta con platformDefault (navegador)
3. Si todo falla, último intento con platformDefault

VENTAJA:
--------
NO depende de `canLaunchUrl()` que puede fallar.
Intenta abrir directamente.
