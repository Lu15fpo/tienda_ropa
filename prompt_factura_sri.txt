*** CONTEXTO TÉCNICO PARA IMPLEMENTACIÓN FACTURACIÓN ELECTRÓNICA SRI ECUADOR ***

ROL: Desarrollador Backend Senior (Node.js/Firebase).
OBJETIVO: Implementar facturación electrónica "Offline" para Ecuador.
STACK: Flutter (Frontend) -> Firestore (DB) -> Firebase Cloud Functions (Backend Node.js).

--- 1. RESPUESTAS A TUS PREGUNTAS DE ARQUITECTURA ---

BACKEND SELECCIONADO: Firebase Cloud Functions (Node.js).
MOTIVO: Seguridad (el certificado .p12 y la firma deben ocurrir en servidor, nunca en el cliente Flutter).
FLUJO:
1. Flutter crea documento en colección 'facturas_por_emitir'.
2. Cloud Function (onCrreate) se dispara.
3. Node.js genera XML, firma, envía a SOAP SRI, recibe respuesta.
4. Node.js actualiza documento Firestore con estado (AUTORIZADO/RECHAZADO) y URL del PDF.
EMAIL: Usaremos 'nodemailer' en la Cloud Function.

--- 2. DOCUMENTACIÓN TÉCNICA EXACTA DEL SRI (AMBIENTE PRUEBAS) ---

A. WSDL (ENDPOINTS SOAP):
- Recepción (Validación y Recepción):
  URL: https://celcer.sri.gob.ec/comprobantes-electronicos-ws/RecepcionComprobantesOffline?wsdl
- Autorización (Consulta de estado):
  URL: https://celcer.sri.gob.ec/comprobantes-electronicos-ws/AutorizacionComprobantesOffline?wsdl

B. VERSIÓN DEL ESQUEMA XML (XSD):
- Versión: 2.1.0 (Formato Factura).
- Codificación: UTF-8.

C. ALGORITMO CLAVE DE ACCESO (49 DÍGITOS):
Formato: [Fecha 8][TipoComprobante 2][RUC 13][Ambiente 1][Serie 6][Secuencial 9][CodigoNumerico 8][TipoEmision 1][DigitoVerificador 1]
- Fecha: ddmmaaaa
- TipoComprobante: "01" (Factura)
- RUC: 1234567890001
- Ambiente: "1" (Pruebas)
- Serie: "001001" (Establecimiento + Punto Emisión)
- Secuencial: 000000001 (9 dígitos)
- CodigoNumerico: 12345678 (8 dígitos aleatorios)
- TipoEmision: "1" (Normal)
- DigitoVerificador: Algoritmo Módulo 11 (Suma ponderada base 7).

D. FIRMA ELECTRÓNICA:
- Estándar: XAdES-BES (Xml Advanced Electronic Signatures).
- Librería recomendada Node.js: 'xml-crypto' o 'xadesjs'.
- Requisito: El archivo .p12 debe desencriptarse y usarse para firmar el nodo <factura>.
