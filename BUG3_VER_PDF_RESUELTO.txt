  🔍 ANÁLISIS DEL PROBLEMA
═══════════════════════════════════════════════════════════════

CAUSA RAÍZ: Desajuste de IDs

PROBLEMA:
---------

1. Al CREAR el pedido:
   - Se genera ID temporal: `UniqueKey().toString()`
   - Ej: "ValueKey#abc123"
   - Este ID se guarda en el campo 'id' en Firestore

2. Al GUARDAR en Firestore:
   - Firestore genera su propio ID automático
   - Ej: "f1qUNZCvrtmE25oNSYVP"
   - Este es el ID REAL del documento

3. Al GENERAR la factura:
   - Se usa el ID REAL de Firestore (correcto)
   - Factura asociada a: "f1qUNZCvrtmE25oNSYVP"

4. Al VER PEDIDOS (fetchUserOrders):
   - OrderModel.fromSnapshot lee `data['id']`
   - Obtiene: "ValueKey#abc123" (ID temporal)
   - NO obtiene: "f1qUNZCvrtmE25oNSYVP" (ID real)

5. Al PRESIONAR "Ver Factura PDF":
   - Busca factura con orderId: "ValueKey#abc123"
   - ❌ NO encuentra (porque la factura está con "f1qUNZCvrtmE25oNSYVP")
   - Muestra: "Factura no disponible"

CAMBIO SIMPLE: Usar `snapshot.id` en lugar de `data['id']`

ARCHIVO: lib/features/shop/models/order_model.dart
LÍNEA: ~109

```dart
factory OrderModel.fromSnapshot(DocumentSnapshot snapshot) {
  final data = snapshot.data() as Map<String, dynamic>;

  return OrderModel(
    id: snapshot.id,  // ✅ CORRECCIÓN: Usa el ID del documento
    userId: data['userId'] ?? '',
    // ... resto del código
  );
}
```

RESULTADO:
  order.id = "f1qUNZCvrtmE25oNSYVP"  ✅ ID real del documento
  snapshot.id = "f1qUNZCvrtmE25oNSYVP"  ✅ Coinciden
